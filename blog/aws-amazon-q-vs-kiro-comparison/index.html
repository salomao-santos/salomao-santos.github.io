Principais Diferenças

IntelliJ IDEA

Força: Melhor para desenvolvimento Java e ecossistema JetBrains

  • Integração nativa com IDEs JetBrains
  • Suporte completo a serviços AWS
  • Project rules com .amazonq/rules/
  • Excelente para Java, Kotlin, Scala

Limitação: Sem custom agents ou features avançadas do Kiro

Visual Studio Code

Força: Versatilidade e inline chat

  • Inline chat (Ctrl+I / Cmd+I)
  • Suporte a múltiplas linguagens
  • Integração com Remote Development
  • Extensão AWS Toolkit completa

Limitação: Sem custom agents ou knowledge base

Kiro CLI

Força: Terminal-first com recursos avançados

  • Custom agents (/agent)
  • Knowledge base com PDFs
  • Tangent conversations
  • Suporte a imagens
  • Autenticação GitHub/Gmail

Limitação: Sem sugestões inline (terminal)

Kiro IDE

Força: Desenvolvimento spec-driven único

  • Spec-driven development completo
  • Agentes integrados e custom agents
  • Checkpointing/rewind de modificações
  • Property-based tests automáticos
  • Código production-ready desde início

Limitação: Ainda em GA, algumas features em evolução

Qual escolher?

Desenvolvimento Java/Kotlin Enterprise

→ IntelliJ IDEA

Melhor integração com ecossistema JetBrains, Spring Boot e serviços AWS.

Desenvolvimento Web/Multi-linguagem

→ VS Code

Excelente para JavaScript, TypeScript, Python, e desenvolvimento full-stack.

Terminal-first / DevOps / Scripting

→ Kiro CLI

Ideal para trabalho em terminal, automação e custom agents.

Novos Projetos / Código Production-Ready

→ Kiro IDE

Melhor para desenvolvimento spec-driven com especificações técnicas detalhadas.
